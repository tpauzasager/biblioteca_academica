2.a Explicar Hashing como método de acceso a datos
	El método consite en tener una función (función hash) que se encarga de calcularle un valor a una clave dada y este valor es la posición en la que toma dicha clave en el indice. Esta función debe tener las siguientes propiedades: Minimizar la colición, Dispeción (pequeños cambios en el valor de entrada genera grandes cambios en el valor de salida). Debido a que el conjunto imagen de la función en menor al dominio, es suceptible a coliciones (que dos valores de entrada den mismo valor de salida), para solucionar esto se cuenta con 3 posibles soluciones: Hashin doble (volver a calcular un valor de salto con otra función hash), Encadenamiento (cada resultado de hash posee una lista asociada en donde se almacenan todas las claves que otorguen dicho valor) y Abierto (se establece un valor del salto que dara en busqueda del próximo espacio disponible). 

2.b Explicar diferencias entre vista y snapshot
	Una vista es aquella que permite al usuario visualizar los resultados de una consulta previamente estipulada, esto es con el fin de ocultar complejidad y dar mayor seguridad. En cambio, la snapshot permite ver al usuario un resultado de una consulta realizada anteriormente y por ende se utiliza para temas de auditorias o generar un histortial, entre otros. La diferencia radica en que, para la vista, la consulta debe ser recalculada cada vez que se la llama, en cambio para la snapshot no y, por ende, no se actualiza.

3.b Realizar una consulta que devuelva el país con el total de sus provincias y ciudades
```sql
SELECT p.nombre, COUNT(pr.idpcia), COUNT(c.idcdad)
	FROM Pais p
		JOIN Provincia pr on p.idpais = pr.idpais
		JOIN Ciudad c on c.idpcia = pr.idpcia
```
