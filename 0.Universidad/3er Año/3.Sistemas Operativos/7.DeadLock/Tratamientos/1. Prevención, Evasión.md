# Prevención
Busca prevenir una de las condiciones necesarias. Restringiendo modo de solicitudes de recursos 
- Muta exclusión: 
- Espera y retención: Los procesos pueden pedir solo al inicio cuales recursos quiere. Si quiere pedir más luego, antes tiene que liberar todos los retenidos 
- Sin desalojo: 

| Condición          | Solución                                                                                                                                                                                                        |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Muta exclusión     | Prevenir muta exclusión sobre recursos                                                                                                                                                                          |
| Espera y retención | - Los procesos pueden pedir solo al inicio cuales recursos quiere <br>- Si quiere pedir más luego, antes tiene que liberar todos los retenidos                                                                  |
| Sin desalojo       | - Si proceso pide recurso bloqueado -> espera y libera todos los recursos que tenga bloqueados<br>- Si proceso pide recurso retenido por otro proceso bloqueado -> Estos son desalojados y asignados al primero |
| Espera circular    | Requerir que los recursos se pidan en orden ![[Pasted image 20240913192549.png]]                                                                                                                                                                    |
___
# Evitar
Proceso debe indicar cant máxima de recursos que puede pedir durante ejecución 
Antes de otorgar recurso, realiza simulación para garantizar que esa asignación no generará un Deadlock

Solo asigna recurso si luego el sistema queda en estado seguro  
- Estado seguro <==> Se pueden asignar todos los recursos a cada proceso en determinado orden, sin que se genere deadlock
