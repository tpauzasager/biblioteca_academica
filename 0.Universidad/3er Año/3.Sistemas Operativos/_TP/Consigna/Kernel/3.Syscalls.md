# Procesos
## PROCESS_CREATE
 Input: 
1. Nombre del archivo de speudocódigo a ejecutar por el proceso
2. Tamaño del proceso en memoria
3. prioridad del hilo main (TID 0)

Funcionamiento:
- Kernel crea PCB y TCB (del TID 0) y deja estado en NEW 

## PROCESS_EXIT
Funcionamiento:
	Finaliza el PCB correspondiente al TCB que ejecutó la instrucción
	Envía todos los TCBs asociados a la cola EXIT.
	Solo puede ser llamado por el TID 0 (del proceso) y se indica a memoria la finalización del proceso
___
# Threads
## THREAD_CREATE
Input:
1. nombre de archivo de pseudocódigo
2. Prioridad del hilo creado
Funcionamiento:
	Se deberá crear el nuevo hilo, generar su TCB con el TID autoincremental
	Poner el hilo en READY
## THREAD_JOIN
Input:
1. TID
Funcionamiento:
	Mueve hilo que lo invoco a BLOCK hasta que TID (pasado por parámetro) finalece.
	Si el TID no existe o ya haya finalizado => Syscall no hace nada e hilo sigue ejecución normal 
## THREAD_CANCEL
Input:
1. TID
Funcionamiento:
	Se pasa el TID (pasado por parametro) a EXIT
	Se avisaa MEMORIA de la finalización
	Al final, hilo que lo ejecuta sigue con su ejecución normal
	Si TID no existe (o ya terminó) => Syscall no hace nada
## THREAD_EXIT
Funcionamiento:
	Finaliza el hilo que lo invocó (lo pasa a estado EXIT)
	Se indica a MEMORIA de la finalización 
___
# Memoria
## DUMP_MEMORY
Funcionamiento:
	Le solicita a MEMORIA (junto al PID y TID) que hace un Dump del proceso
	Bloquea el módulo que la invocó hasta que memoria confirme la finalización 
	Si hay error => Se pasa proceso a EXIT
