# Métodos de planificación  
## FIFO
Según orden de llegada a READY
## Prioridades
Se elegirá el siguiente hilo según el número de prioridad más bajo (encaso de empate => FIFO)
Desalojo: 
	Llega a cola READY un hilo con mayor prioridad al que se está ejecutanto => Se desaloja el que se está ejecutando y se pasa a ejecutar el que llegó 

## Multinivel
Se tiene una cola para cada nivel de prioridad existente entre hilos
Algoritmo entre colas:
	Prioridad con desalojo (el explicado antes)
Algoritmo en cola:
	RR con Q definido en archivo configuración 
	Al terminar el Q, se coloca al final de la cola
Desalojo entre colas:
- Al llegar un nuevo hilo con mayor prioridad, se desaloja al que se está ejecutando y se lo trata como si hubiese terminado el Q


# Ejecución 
Se envía al CPU el TID y PID
Queda a la espera de recibir un TID después de la ejecución, junto al motivo por el cual fue devuelto

Desalojo:
	Se envía interrupción al CPU a traves de syscalls (invocando la API explicada entes)

Al recibir TID del hilo en ejecución, si esto requiere replanificación, se elige el siguiente hilo a a ejecutar. Durante este período, la CPU queda esperando
