Objetivo: 
	Gestionar peticiones a la memoria para creación y eliminación de procesos e hilos
___
# Creación de proceso
Cola NEW (FIFO)

Funcionamiento: 
	Cuando llega proceso a la cola NEW, se pide a MEMORIA reservar el espacio requerido
	Si hay espacio => Se crea el TID 0 (del proceso) y se pasa a estado READY (se repite para el siguiente proceso)
	Si no hay espacio => Se espera hasta que otro proceso finalice 

# Finalización de proceso
Funcionamiento:
	Kernel informa a MEMORIA sobre la finalización, cuando recibe confirmación, libera su PCB e intenta inicializar otro proceso es cola NEW (si es que hay)

# Creación de hilos
Funcionamiento:
	Informar a MEMORIA e ingresar a cola READY correspondiente (según nivel de prioridad)

# Finalización de hilos
Funcionamiento: 
	Informar a MEMORIA y mover a estado READY a todos los hilos bloqueados por este hilo (por THREAD_JOIN y por mutex del hilo finalizado)
