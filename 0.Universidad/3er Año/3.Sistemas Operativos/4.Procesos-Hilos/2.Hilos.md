# Definición
- Unidad básica de utilización del CPU
- Conjunto de registros y espacio de pila
- Comparte código, datos y recursos con hilos pares (del mismo proceso), por lo que no necesitan al SO para comunicarse
- Cada proceso está formado por uno (el main) o más hilos
- Permiten paralelismo dentro de un proceso 
- No hay protección entre hilos

# Hilos vs Procesos
- Comunicación entre hilos sin necesidad del SO
- Más eficiencia en creación y cambio (switch) entre hilos
- Cuando Proceso mueren sus hilos tambien lo hacen
- Muerte de un hilo puede afectar al resto del proceso (por que comparten recursos)
- Muerte de un proceso no afecta al resto de los procesos pares 
- Procesos son más seguros, porque no comparten recursos

# KLT vs ULT
| KLT                                                                                                                                                                              | ULT                                                                                                                                                                                                                               |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| - Hilos a nivel Kernel<br>- Nativos                                                                                                                                              | - Hilos a nivel Usuario<br>- Green Threads                                                                                                                                                                                        |
| - SO conoce su existencia y controla su ejecución<br>- Conmutación de contexto más rápida <br>- **Permiten paralelismo entre hilos pares**<br>- Implementan propia planificación | - Gestión por bibliotecas en modo usuario (SO no los conoce)<br>- Bajo overhead<br>- Cambio más liviano de hilo<br>- **No permite paralelismo**<br>- Si uno realiza una operación bloqueante, los demás pares no pueden continuar |
