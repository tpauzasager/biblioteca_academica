# Definición
- Programa en ejecución (incluye más que el código del mismo)
- Programa -> Entidad pasiva    ;    Proceso -> Entidad activa 

# Estructura 
- Modelo PCB (Process Control Block)
- Se guarda en Memoria RAM
![[Pasted image 20240830230346.png]]

## Stack
- Se guardan los datos temporales que utiliza el proceso (variables, retornos, registros, etc)
- Sistema LIFO
## Heap
- Acceso random
- Para persistir datos que se usan más allá de una función 
- Suele ser más grande que el Stack
## Data
- Para variables globales, estáticas o constantes (ya que tienen una ubicación fija en la memoria)
___
# Creación de procesos
- A través de una syscall (fork) se clona el proceso
## Cuando padre crea hijo/s:
- Continúa ejecución concurrentemente con el hijo
- Espera a que todos o algunos de los hijos finalice 
## Espacio en memoria
- El hijo, al ser copia exacta del padre, tienen mismo PCB
- Si hijo modifica algún dato, se asigna nuevo espacio en memoria solo para ese dato (así no es necesario copiar todo el PCB)
___
# Finalización
- Última operación del proceso -> Llamada al SO 'exit' (para que lo elimine)
- Se envía a su padre la información de salida y se liberan recursos (eliminando también sus hilos)
- Padre puede terminar ejecución de hijos ([[0.Universidad/3er Año/3.Sistemas Operativos/999.Aclaraciones|Aclaración 2]])
___
# Comunicación entre Procesos
- Procesos no afectan ni puede ser afectado por el otro ==> Independientes
- Procesos comparten recursos ==> Cooperativos
