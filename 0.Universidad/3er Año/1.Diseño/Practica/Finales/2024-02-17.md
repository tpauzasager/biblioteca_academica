3. (15 puntos) Con el objetivo de tener una comparación del valor de lo que estamos pagando
nosotros y los valores de mercado, se pretende integrar el Sistema con una API de Precios de
Referencia. Sabiendo que dicho Sistema cobra por cada solicitud que realicemos; ¿cómo podría
minimizar los gastos mensuales pero seguir utilizando este Servicio? Nombre y explique el
funcionamiento del mecanismo/componente propuesto.
	- Inicialmente implementaría test mockeando esta API, así no será necesario pagar por cada vez que se testee el sistema.
	  Por otro lado, las respuestas recibidas las agregaría auna cache, la cual será actualizada todos los días (o semanas dependiendo de la situación del país). Esto es así para poder minimizar la cantidad de requests que se realizan a la API, ya que, muchas de las consultas podrán ser resueltas consultado la cache. Además, esto no consumira muchos recursos, ya que, no se requiere una gran cantidad de datos para cada producto, por lo que la cache no llegaría a ser de gran tamaño.
