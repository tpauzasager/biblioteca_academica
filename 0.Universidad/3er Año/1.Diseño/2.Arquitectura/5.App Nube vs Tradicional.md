Aplicativos en la Nubel:
	- Cambia forma en que se diseñan apps
	- No es monolitica, se descompone en **servicios menores** y descentralizados, que se comunican por API (o uso de eventos o mensajería asincrónica)
	- Apps se escalan horizontalmente
.
	- Estado de aplicaciones se distribuye 
	- Operaciones realizadas en paralelo asincrónicamente
	- Apps deben ser resistentes a errores 
	- Implementación automatizada y predecible 
	- Supervisión y telemetría para obtener visión gral del sistema 

# Diferencias
Generalmente...

| Caracteristica  | Local - Tradicional                             | Nube - Moderna                         |
| --------------- | ----------------------------------------------- | -------------------------------------- |
| Arquitectura    | Monolítica                                      | Descompuesto (No monolíticos)          |
| Escalabilidad   | Predecible                                      | Elástico (a demanda)                   |
| Persistencia    | BD Relacional                                   | Poliglota (combinación de tecnologías) |
| Procesamiento   | Sincronizado                                    | Asincrónico                            |
| Errores         | Para evitar errores                             | Para que falle y poder manejarlos      |
| Actualizaciones | Grandes ocacionales                             | Pequeñas y frecuentes                  |
| Administración  | Manul                                           | Automatizada                           |
| Infraestructura | Servidores de copo de nieve (escalado vertical) | Inmutable                              |
