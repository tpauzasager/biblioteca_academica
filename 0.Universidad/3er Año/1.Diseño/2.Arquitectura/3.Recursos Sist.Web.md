# Protección de Recursos
Niveles:
	1. Autenticación 
	2. Autorización basada en roles
	3. Autorización basada en roles y permisos

Autenticación 
	- Requiere que usuario se identifique antes de acceder a recursos
	- Con vista -> Redirigir a página log in
	  API rest -> Devolver 401 (no autorizado)

Roles
	- Depende de protección por Autenticación 
	- Requiere que usuario (previamente identificado) tenga un rol particular para acceder a recurso
	- Permite que no todos los usuarios manipulen los mismo recursos

Roles y Permisos
	- Depende de protección por Autenticación 
	- Requiere que usuario tenga rol particular y que, dicho rol, tenga permisos necesarios 
	- Permite que usuarios con distintos roles cuanten com mismo permisos
	- A veces hace falta pensar en **alcances** del permiso sobre los rescuros. 
	  Ej: Encargado puede ver a los empleados, peros solo los que tiene a su cargo
