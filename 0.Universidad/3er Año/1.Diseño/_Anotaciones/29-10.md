# Integración de sistemas
Definición:
	- Establecer comunicación entre ellos
	- Compoartir datos y funcionalidades 
	- Aumentar la cohesión evitando duplicación de lógica 

Porque integramos los sistemas:
	- Los sistemas no son aislados (necesitan de otros para subsistir)
	- Hay lógica ya creada => no hace falta recrearlos
	- Sistemas deben compartir datos sin copiarlos (consultandolos)
	- Necesidades del negocio/marco legal

Sincrónica o Asincrónica?
	- Depende exclusicamente de necesidades de negocio y restricciones técnicas 
	- Tambien hay que ver el sincronismo del proceso mismo. Capaz la interacción con el usuario es sincrónica, pero por detrás, el procesamiento, es asincrónica

## SOAP
- Bajo protocolo SOAP (Simple Object Access Protocol)
- SOAP -> Protocolo con formato XML
- Puede funcionar bajo protocolo HTTP
	- Tambien por FTP, POP3, SMTP, TCP, Colas de mensajería
- Usa WSDL (Web Service Description Lnaguage) dond eexpone la interfaz y será su punto de entrada
  Es un archivo wsdl, accesible por la red mediante url (con terminación ?wsdl)
  En archivo wsdl se describen todos los servicios web y operaciones disponibles y las estructuras de datos de los mensajes de cada operación (xsd)
  En archivo xsd se describen estructuras de datos de los mensajes de cada operaicón

## gRPC
- **Framework** de llamadas a procedimientos remotos 
- Moderno open source de alto rendimiento que puede ejecutarse en cualquier entorno 
- Puede conectar servicios eficientemente en y entre centros de datos con soporte de balanceo de carga, ratreo, verificación de estado y autenticación 
- Tambien para conectar dispositivos, aplicaciones móviles y navegadores a servicios backend

- Desarrollado en 2015 por Google
- Flujo de datos HTTP/2
- Estructura y distibución de datos mediante Protocol buffer
- Flujo transmitidos en binarios compactados surgen en serialización y deserialización 

### Protocol Buffer
Formato de intercambio de datos (equivalente a json)
Se almacenan en binario
Se combierten antes de pasar a binario para que pueda ser leído por diferentes lenguajes 

### Implementación

2. Generar código fRPC del archivo .proto
3. Implementación del servidor (en lenguaje deseado)
4. Creación del stub del cliente (que llama al aservicio)

## GraphQL

- Das poder a cliente de pedir exactamente lo que necesita y nada más 

- Solo hay un endpoint
- Cliente puede depurar respuesta
