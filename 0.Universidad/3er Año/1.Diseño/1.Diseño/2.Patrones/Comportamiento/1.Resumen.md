| Nombre          | Que hace?                                                                                                                                              | Cuando usarlo?                                                                                                                                                                                                 | Beneficios                                   |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| State           | Una clase por cada posible estado que puede tener un objeto<br>Transiciones entre posibles estados                                                     | Compotamient del objeto depende del estado                                                                                                                                                                     | Cohesión<br>Mantenibilidad<br>Extensibilidad |
| Template Method | Define esqueleto de algoritmo (pasos que si o si se deben realizar)                                                                                    | Varias clases tienen mismos pasos y orden para realizar algo<br>Usar polimorficamente objetos para ejecutar mismo algoritmo (pero con diferentes implementaciones)                                             | idem                                         |
| Strategy        | Encapsula distintas formas de resolver mismo problema en diferentes clases<br>Permite intercambiar forma de resolver un problema en runtime            | Objeto realice misma acción pero con algoritmos diferentes <br>Muchas formas de realizar misma acción (con diferentes pasos) en el mismo objeto<br>Configurar la forma en que objeto realiza acción en runtime |                                              |
| Adapter         | Encapsula uso de la clse que se quiere adaptar en otra clse que concuerda con la interfaz requerida                                                    | Se quiere usar una clase existente, pero interfaz no coincide o es desconocida                                                                                                                                 |                                              |
| Observer        | Notificar interesados los eventos que le ocurren al objeto en el que están interesados<br>Desacopla acciones a ejecutar de los eventos que los activan | Disparar acciones de diferentes naturalezas frente a un mismo evento<br>Notificar a objeto de un evento sin necesidad de conocerlo                                                                             |                                              |
