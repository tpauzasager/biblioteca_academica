# Tips
- Diferenciar estimaciones, objetivos y compromisos 
- Asociar % de confiabilidad a estimación 
- Estimación == Rango (no valor único)
- Presentar supuestos que se tuvieron en cuenta para estimar 
- **Ley de Parkinson**: "Toda tarea se expande hasta ocupar el tiempo que tiene asignado"
- Considerar todas las actividades relacionadas al desarrollo 
- Usar **datos históricos**

# Creíbles 
1. Las **estimaciones las hacen las personas**, no herramientas ni modelos
2. Se necesitan **juicios razonables** y **compromisos** con los objetivos organizacionales que no pueden delegar a modelos automáticos.
3. Las estimaciones se **basan en comparaciones**
4. Se necesita **historia de proyectos pasados**
5. Si necesitamos pedir prórrogas cae nuestra credibilidad.
6. Para una estimación correcta **no se usa un solo método**
