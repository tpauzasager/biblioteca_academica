# Definición 
- Variación de "Wideband Delphi"
- Estimación de tamaños relativos entre las tareas 
- Bueno para proyectos incrementales a largo plazo 

# Reglas 
| Nombre                 | Definición                                                                                             |
| ---------------------- | ------------------------------------------------------------------------------------------------------ |
| Participantes          | Todo el equipo tiene que participar                                                                    |
| Timebox                | Fijar cant de tiempo que se va a estimar                                                               |
| Priorizar              | Requerimientos leidos y priorizados por el equipo                                                      |
| Escala                 | Medición en **Story Points** que sigue escala de Fibonacci                                             |
| Tarea Pivote           | Seleccionar una tarea significativa y asignar SP<br>Esta luego se usa para relacionar con demás tareas |
| Velocity               | Determinar cuanso SP hacen un sprint<br>Basandome en sprints anteriores                                |
| Definición "Terminado" |                                                                                                        |

# Métodología 
1. Moderador elige User Story (requerimiento) 
2. Equipo estima (asigna SP al US)
3. El menor y mayor debaten por su elección 
4. Se da oportunidad a cambiar la respuesta a los demás 
5. Repetir hasta completar los SP que hacen a un sprint 

# Ventajas
- Se estima por **tamaño** => Si equipo cambia, tarea no se modifica
- Al ser sprint corto => se corrigen desvíos rapido 
- Poco tiempo invertido en la planning 

# Cuando usar? 
- Proyectos **con experiencia** 
- Proyectos **largos**
