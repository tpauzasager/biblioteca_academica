# Definición
- Método para dividir el proyecto en partes pequeñas y comprensibles 
- Se usa medir **que hace** el sistema, no como lo hace 
- Function Point (FP): Unidad de medida para las funcionalidades del sistema 
- **Generar puente entre equipo técnico y mundo comercial** 
- Para **sistemas con varias tecnologías** 
## Componentes
**Input** -> Componentes (requerimientos)
**Output** -> Function Points

# Objetivos 
- Medir sw desde **perspectiva del usuario**
- Permitir comparación (independientemente de técnología usada)
# Metodología 
1. **Determinar componentes** del producto y categorizarlos
2. **Asignar** niveles de **complejidad** 
3. Calcular **UFP** (function points no ajustados)
4. Caclular **factor de ajuste**
5. Calcular **NFP** (function points neto)
# Beneficios 
- Medir tamaño objetivamente 
- Permitir ententendimiento entre equipo desarrollo y usuarios (no técnicos)
# Cuando usar? 
- Comparar tamaño/productividad con equipos con otras técnologías 
- Benchmarking 
- Justificar costos
