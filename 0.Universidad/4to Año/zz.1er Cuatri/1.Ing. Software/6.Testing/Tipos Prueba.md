# Tipos

## Unitaria 
- Sobre una unidad de código (claramente definida)
- Realizado por devs 
- Basado en diseño detallado 
- Más cohesión => Más sencillo probar 
## Integración 
- Probar que partes de sistema (que funcionan bien aisladamente) tambien lo hagan en conjutno 
- Tipos
	- No Incremental -> Big Bang
	- Incremental       -> BOTTOM-UP ; TOP-DOWN 
## Aceptación de Usuario
- Realizada por usuarios 
- Para verificar que el sistema cumple requerimientos 
## No funcionales
- Para requerimientos no funcionales 
Tipos

| Tipo        | Definición                                      |
| ----------- | ----------------------------------------------- |
| Volumen     | Volumen máximo de requests                      |
| Performance | Sist soporta tiempos de respuesta               |
| Stress      | Someter sistema excediendo limites de capacidad |
| Seguridad   |                                                 |
| Usabilidad  |                                                 |
| Otros       |                                                 |
## De Regreción 
- Verificar que las cosas que funcionaban antes de la modificación sigan funcionando 
- Probar lo viejo 
## De Humo
Smoke Test
- Verificar muy rápidamente funcionalidad básica del sistema 
## Alfa y Beta
- Entregar primera versión al usuario para que la pruebe
- Alfa -> En entorno controlado 
- Beta -> En entorno del usuario
