# Introducción
- Pruebas no mejoran SW, solo muestran cuantas fallas tiene
- Buen diseño beneficia pruebas, corrección y mantenimiento
- No probar no elimina errores, acorta tiempos ni abarata costo

# Enfoque de Pruebas
- Caja blanca importante complemento a caja negra 
	Para generar mejores casos de prueba
 - Caja gris prueba sw como si fuera caja negra, pero agregando casos por conocimiento del sw (caract de caja blanca)
## Caja Negra 
- Ninguna técnica de generación de condiciones y casos es completa 
- Cada técnica ataca distinto problema (es conveniente combinar varias)
- Tener en cuenta conjetura de errores 
# Exploratory Testing
- Forma de encarar testing 
- Dependiente del tester
- Depende de conocimiento adquirido durante la ejecución 
# Automatización
- Acumular experiencia en testing manual 
- Setear expectativas
- No esperar resultados inmediatos 
- No automatizar todo
- No depender de una sola herramienta 
- Separar equipo de automatización y ejecución 
- Analizar cada proyecto en particular
