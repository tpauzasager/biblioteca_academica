# Definición 
- Administración visual del flujo de trabajo 
- **Limitación del WIP**
# Objetivo  
- Detectar **cuello de botella** 
- Evitar abrumación 
- Mejorar comunicación (evitar duplicación de trabajo)
# Cuando usar
- **Muchas tareas** para realizar constantemente 
- Proyectos inestables (**muchos cambios** en requerimientos)
# División 
| TO DO                  | WIP                                                                                       | DONE |
| ---------------------- | ----------------------------------------------------------------------------------------- | ---- |
| Lo que falta por hacer | Lo que se está haciendo ahora<br>Una persona solo puede tener una tarea asignada a la vez |      |
# Métricas 
- Touch time -> Tiempo neto de trabajo sobre una tarea
- Lead time -> Tiempo entre momento de pedido y entrega para un item
- Cycle time -> Tiempo entre inicio y fin del proceso para un item
