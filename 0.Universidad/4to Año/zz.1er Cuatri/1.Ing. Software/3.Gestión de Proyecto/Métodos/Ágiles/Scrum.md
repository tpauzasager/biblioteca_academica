# Definición 
- Marco de trabajo para administración de proyectos 
- Enfatiza trabajo en equipo 
- Proceso **iterativo incremental** hacia objetivo definido 
- **Comenzar con lo que puede ser visto. Luego, seguir el avance y modificar lo necesario**

# Framework 
- **Sprint:** período fijo de tiempo (1-4 semanas)
- En cada sprint, equipo contruye y entrega incremento de producto 

# Elementos 
## Roles
| Nombre        | Definición                                                                                  | Función                                                                 | No función                                                         |
| ------------- | ------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- | ------------------------------------------------------------------ |
| Product Owner | Representa a clientes y usuarios<br>Dueño del producto / BL                                 | Definir requerimientos y prioridades<br>Establecer plan de entregas<br> |                                                                    |
| Scrum Master  | Dueño del proceso<br>Colabora con equipo                                                    | - Organizar y eliminar impedimentos<br>- Protege y cuida al equipo      | - Asignar tareas a devs<br>- Determinar que entregar en sig sprint |
| Equipo        | Dueño de procesos de producción/ingeniería<br>Pequeño<br>Interdiciplinario y Autogestionado | Transforma el BL en incrementos funcionales                             |                                                                    |

## Ceremonias 
| Nombre                     | Definición                                                                                                                         | Integrantes                                  | Frecuencia                     |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- | ------------------------------ |
| Sprint Planning            | Que se va a trabajar durante el sprint? (del BL)<br>Cantidad de trabajo para realizarlo? (story points)                            | Todo el equipo                               | **incio** del sprint           |
| Daily Meeting              | Que hice hasta ahora (en el sprint)?<br>Que voy a hacer despues?<br>Que me frena?                                                  | equipo y scrum master<br>PO (puede no estar) | por día                        |
| Sprint Review              | Se muestras funcionalidades desarrolladas <br>Se obtiene feedback del negocio                                                      | Todos                                        | **final** del sprint           |
| Retrospectiva              | Revisar:<br>- Cosas que funcionan bien<br>- Cosas que queremos que funcionen mejor<br>Propuesta de mejoras<br>Lecciones aprendidas |                                              | final del sprint               |
| Product Backlog Refinement | Reestimación de SP del BL                                                                                                          |                                              | Espontanea (cuando se necesta) |
## Artefactos 
| Nombre                 | Definición                                                                            |
| ---------------------- | ------------------------------------------------------------------------------------- |
| Product Backlog (BL)   | Conjunto de requerimientos a desarrollar en el proyecto<br>Ordenados segurn prioridad |
| Sprint Backlog         | Requerimientos que se van a implementar en el sprint actual                           |
| Incremento de Producto | Lo que se tienen del producto hasta la fecha                                          |
