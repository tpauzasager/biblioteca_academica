Escalabilidad y Disponibilidad de almacenamiento onPremises 

Nivel de RAID:
	- Cantidad de veces que los bytes están duplicados 
	- Aumenta redundancia de datos => Más tolerancia a fallos 
	RAID 0:
		- Todos los discos son independientes entre si (no hay redundancia)
	RAID 1: 
		- Una duplicación de datos (un disco depende unicamente de otro)
		- Dentro del mismo dispositivo
# Almancemiento de datos 
## Arquitecturas 
### DAS
Sistema de almacenamiento accedido de forma directa 
Servidor (con propio OS y FS) que esta conectado directamente a conjunto de discos 

Ventajas:
	- AB garantizado (porque la conexión es por física)
	- Fácil configuraicón y mantenimiento 
Desventajas:
	- Escalabilidad limitada a cant de puertos para disco 
	- Accesible solo para dispositivos conectados 
### SAN
Red de Almacenamiento de Datos 
Red de procesamiento conectada a red de almacenamiento (a travez de switches SAN)

Redundancia de:
	- Switches
	- Conexiones a switches (de cada servidor)
Ventajas:
	- Gran AB (usa FO)
	- Recursos accesibles a todos en la red
Desventajas:
	- Mayor costo por componentes y redundancia de componentes
	- Mayor complejidad de configuración y mantenimiento

### NAS
Almacenamiento conectado en red 
Uso hogareño 

Ventajas:
	- Conexión de arquitectura de red de datos existente (me conecto por canales que ya existen)
	- Acceso concurrente 
	- Administración delegada a dispositivo NAS
Desventaja:
	- Menor rendimiento (por protocolos usados para transferencia)
### Almacenamiento distribuido (Horizontal)
Definición:
	- Se replican componentes
	- Repica información en dispositivos separados 

Almacenamiento cucaracha 

Ventajas:
	- Mayor disponibilidad (duplicación de datos distribuido)
	- Mayor escalabilidad
	- Menor costo de HW
Desventaja:
	- Mayor complejidad de configuración 
# Requerimientos 
Disponibilidad:
	- Se da a través de redundancia de dispositivos (de almacenamiento, administrativos o de datos) 
	- Tipos:
		- Online -> Recuperación inmediata 
		- Offline -> Recuperación diferida (requiere más tiempo para levantar los servicios)
Rendimiento:
	Métodos de medición:
		- IOPS -> input output per second (Base de datos)
		- MB/sec -> Thoughput per second (Streaming media)
		- Response Time (ms) -> Tiempo que tarda en enviar todos los datos de la request
