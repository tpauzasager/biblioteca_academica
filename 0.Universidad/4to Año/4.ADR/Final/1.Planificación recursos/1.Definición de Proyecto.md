# Definiciones
PMI -> Esfuerzo  temporal comprmetido con creación de producto o servicio 
PRINCIE -> Ortanización temporal 

Proyecto vs Producto
	Proyecto tiene fecha inicio, desarrollo y fin
	Producto prevalece más allá

Enfoque a medida (diamante)
	- Aplicación de gestión de proyectos flexible y adaptable 
	- Cuando -> Alto nivel incertidumbre y complejidad
## Fases 
Ciclo PDCA :
1. Plan -> Identificar y analizar problema
2. Do -> Elaborar e implementar solución
3. Check -> Evaluar resultados
4. Act -> Estandariar solución y capitalizar oportunidades

Fases :
1. Pre-proyecto -> Determinar factbilidad técnica y económica del proyecto (para ver si se puede hacer y si vale la pena)
2. Inicio -> Se infoma a todos los involucrados y se elabora plan 
3. Ejecución -> Desarrollo del plan 
4. Cierre -> Comprobar tareas finalizadas e informar 
## Riesgos y Problemas
Riesgo -> Evento o condición incierta que, si ocurre, impacta en los objetivos del proyecto

Atributos
	$Severidad = Prob\space ocurrencia*Impacto$ 

Gestión : Ocurre durante toda la vida del proyecto
	1. Identificación -> Reconocer fuentes de riesgo y consecuencias
	2. Análisis -> Determinar necesidad
	3. Tratamiento
		1. Evitar -> Eliminar por completo el origen o el area de efecto
		2. Aceptar -> Nada
		3. Transferir -> Contratar tercero para solución (no transfiere responsabilidad)
		4. Mitigar -> Reducir riesgo (pero no anularlo)

Contingencia
	- Plan de recuperación antes desastres (para cuando ya ocurrió)
	Partes
		- Prevención -> Para restituir el servicio
		- Emergencia -> Para durante la materialización (paliar efectos)
		- Recuperación -> Para restaurar el estado anterior
Mitigación 
	- Prevenir o reducir riesgo antes de que ocurra
## Alcance
Alcance -> Definición exacta y unívoca de todo lo que estára comprendido dentro del proyecto

Documento de Alcance
	- Guia del equipo 
	- Linea base para ver si cambios están o no fuera de los límites 
	- Base para estimar esfuerzo y duración 
	- EDT (Estructura de Desglose de Trabajo) de las partes de proyecto
	Componentes
		- Descripción de objetivos
		- Justificación del proyecto 
		- Listado de requerimientos y entregables
		- Definición de límites y restricciones
		- Descripción de supuestos
		- Hitos -> Punto de control; Entregable <=> Hito & Hito =/> Entregable
		- Dependencia con otros proyectos
