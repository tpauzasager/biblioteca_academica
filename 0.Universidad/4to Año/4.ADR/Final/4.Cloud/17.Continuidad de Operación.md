# RTO
Recovery Time Objective
	Tiempo que tarda una infa en volver a estado disponible 
# RPO
Recovery Point Objective 
	Cantidad de datos que la empresa va a perder por el fallo (tiempo desde el último punto de guardado hasta la ocurrencia del incidente)

# BIA
Business Impact Analisis
	Modelo de niveles
		1 - Aplicaciones esenciales qu requieren RTPO en menos de 15min
		2 - 2h < RTO < 4h
		3 - 4h < RTO < 24h
# Replicación de Datos

|                           | Ventajas                                                                                              | Desventajas                                                                                                        |
| ------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Discos locales/externos   | - Rápido acceso<br>- Integración con app y BD                                                         | - Alto costo<br>- No transportable<br>- Si falla en todo el datacenter => pierdo datos                             |
| Cintas                    | - Bajo costo<br>- Transportable                                                                       | - Mayor tiempo de recuperación                                                                                     |
| Sitio externo             | - Resguardo fuera del datacenter                                                                      | - Costo de licencia de replicación (opex)<br>- Ante contingencia, no puedo seguir operando en datacenter principal |
| Datacenter Externo (DRS?) | - Ante contingencia, se puede seguir usando el datacenter secundario<br>- Bajo tiempo de recuperación | - Alto costo (duplicación de infra)<br>- Costo en licencias de replicación                                         |

DRP == Plan de recuperación de datos
