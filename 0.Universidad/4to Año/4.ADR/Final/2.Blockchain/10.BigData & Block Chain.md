# Big Data
Definición 
	Conjunto de datos extensos

Caracteristicas (5 Vs):
	- Volumen
	- Variedad -> diferentes formatos de datos (estructurados, no estructurados, etc)
	- Velocidad -> alta vel de acumulación y necesidad de vel de procesamiento
	- Veracidad -> precisión e integridad en generación y procesamiento de datos
	- Valor -> valor de los datos

Casos de uso:
	- Experienca de usuario 
	- Desarrollo de productos
	- Mantenimiento predictivo
	- Eficiencia operacional 
	- Faude y conformidad


Data engineering
	Para superar cuellos de botella en procesamiento de datos y manejo de datos para aplicación de big data
Data science
	Encontrar información procesable a partir de grandes conjunto de datos
Data analytics
	Análsisis detallado de datos para tomar decisiones informadas
	Analisis en lo micro
## Tecnologías

| Nombre                        | Definición                                                                |
| ----------------------------- | ------------------------------------------------------------------------- |
| Data Lake                     | Enorme repositorio de datos recopilados de diferentes fuentes             |
| Data Warehouse                | Data Lake pero con procesamiento previo al almacenamiento estructurado    |
| NoSQL DB                      | Almacenar datos no estructurados y alto rendimiento                       |
| Predictive analytics          | Pronostica eventos o comportamientos futuros (basado en datos históricos) |
| Big data security solutions   | Seguridad para repositorios de Big data                                   |
| Big data governance solutions | Para disponibilidad, usabilidad e integridad de los datos                 |
| Streaming analytics           | Análisis de datos actuales (en tiempo real)                               |
| Edge computing                | Opuesto a cloud computing. Analisan datos creados muy cerca de ellos      |
## Seguridad
Capas:
	1. Datos entrada -> Proteger trásito desde la fuente a la plataforma
	2. Datos almacenados -> tecnologías para almacenado seguro
	3. Datos salida -> Cifrar y no envíar datos protegidos por regulaciones

Tecnologías:
	- Cifrado
	- Control de acceso
	- Detección y prevención de instrusos
	- Seguridad física
	- Gestión centralizado de claves

# Blockchain
Definición 
	Red P2P independiente de una entidad centralizada para llegar consenso
	Registro distribuido donde cada par tiene propia copia del registro

Principios
	Integrada en la red
	Poder distribuido
	Valor como incentivo
	Seguridad
	Privacidad 
	Preservación de derechos
	Inclusión

Pasos
	1. Usuario solicita transacción 
	2. Se crea bloque que representa transacción 
	3. Bloque se difunde a todos los nodos de la red
	4. Todos los nodos validan bloque y transacción 
	5. Bloque se añade a la cadena 
	6. Transacción se veririca y se ejecuta 

Usos
	- Contratos
	- Libros contables/de registro
	- Historial clínico
	- Escrituras

Ventajas
	- Naturaleza distribuida 
	- Estabilidad
	- Sistema trustless -> No requiero confiar de terceros
	- Incorruptible
	- Mayor velocidad
	- Transparencia 
	- Trazabilidad
	- Libre de errores

Desventajas
	- Ataque del 51%
	- Modificación de datos
	- Calves privadas 
	- Ineficiente
	- Almacenamiento 
	- Apertura
