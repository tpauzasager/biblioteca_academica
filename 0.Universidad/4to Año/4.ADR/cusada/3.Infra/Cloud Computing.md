# Definición 
- Gestión de todo los recursos (puede ser bueno o malo)
Conjunto compartido de recursos que pueden ser accedidos a traves de internet
Escalable a demanda

Caracteristicas
	1. **Autoservicio bajo demanda** -> Clientes acceden a recursos automaticamente
	2. **Amplio acceso a través de la red**
	3. **Conjunto compartido de recursos** -> Recursos compartidos entre clientes
	4. **Rápida elasticidad**
	5. **Servicio medido** -> Controlan y optimizan recursos asignados a cada cliente

# Modelos de Despliegue 
**Private** -> Proporcionada y consumida por misma organización
**Public cloud** -> Para uso del publico en gral
**Community** -> Para uso exclusivo de comunidad de consumidores 
**Hybrid** -> Composición de dos o más

# Datacenter vs Cloud
Datacenter -> Gestión de todo los recursos (puede ser bueno o malo)

|            | Pros                                                                                                                               | Cons                                                                                                                        |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Datacenter | - Puede ser por acceso local<br>- Manejo propio de seguridad                                                                       | - CapEx (capitalización) elevado<br>- OpEx (pago por operación) atado a capacidad instalada<br>- Tiempo y costo de escalado |
| Cloud<br>  | - CapEx = 0<br>- OpEx se ajusta a demanda<br>- Flexibilidad para escalabilidad<br>- Uso de estandares<br>- Actualización constante | - OpEx proporcionalmente más elevado<br>- No se puede gestionar seguridad<br>- Se comparte infraestructura                  |
