1. Conmutación
	Circuitos:
		- Monpolización 
		- Sin congestión 
		- Transmisión continua 
		- Ruta dedicada 
		- Mejor para voz
	Paquetes:
		- Sin monopolización 
		-Transmisión por paquetes (fragmentos del mensaje original)
		- Mejor para datos
		Tipos
			Circuito virtual
				- Orientado a la conexión
				- Congestión solo al establecer conexión
				- Ruta no dedicada
			Datagrama
				- No orientado a la conexión
				- Congestión en todo momento
				- Sin ruta (se elige mejor camino en el momento)

2.  x.25:
	- Capas 2 y 3
	- Conmutación 
		- Capa 3 -> Datagrama
		- Capa 2 -> Trama (circuito virtual)
	- Control de errores (si se trabaja en modo circuito virtual) => Permite trabajar con canales con alto BER
	- PDU -> Trama o Datagrama (dependiende de que modo se use); Tamaño variable
	FR:
	- Capa 1 y 2 
	- Conmutación en capa 2
	- Con control de errores, pero sin corrección ni retransmisión => Se trabaja con canales con bajo BER
	- Orientado a la conexión por circuito virtual permanente
	- PDU -> Trama; Tamaño variable
	ATM:
	- Capa 1, 2 y 3 (AAL)
	- Conmutación en capa 2
	- Sin corrección ni detección de errores
	- PDU -> celda (52B); Tamaño fijo

3.  Ambos son enrutadores dedicados a su respectiva tecnología WAN
	PAD:
	- Para x.25
	- En capa 3
	- Para comunicar canal asincronico con uno sincronico 
	- Adaptador para x.25 (DTE <-> DCE)
	FRAD:
	- Para Frame relay
	- En capa 2
	- Para comunicar canales sincronicos 
	- Adaptador para FR (CRE <-> POP)
